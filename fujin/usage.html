FUJIN 使い方ガイド

花岡車輌の生産管理を「全社で見える化」するためのダッシュボードです。
SMILE V生産革新のデータを毎朝取り込み、判断に必要な情報だけを整理して表示しています。

はじめに

FUJINとは

SMILEのデータは正確ですが、画面を行き来しないと全体像が見えにくいという課題があります。 FUJINは、SMILEから日次で出力したCSV/TXTを再構成し、班長クラスが「いま何を判断すべきか」を1画面で確認できるようにするためのプロトタイプです。

位置づけ： FUJINは判断補助のためのビューです。データの正本はSMILEにあります。 SMILEで作業した結果が翌朝のFUJINに反映される、という関係です。

共通の前提

① データ更新タイミング

時刻処理
毎朝 3:00RPAがSMILEから最新CSVをSharedMastersに出力
毎朝 4:00FUJINが再ビルドされ、画面表示が最新に更新

つまり、FUJINで見ている数字は「その日の朝3時時点のSMILE」です。
画面右上「データ基準日」と、品目詳細パネル内「現在庫: YYYY/MM/DD基準」で常に基準日を確認できます。

3日以上古いデータを表示している場合、「⚠ N日前」のバッジが現れます。RPAが止まっている可能性があるので、システム担当に連絡してください。

② 認証

花岡車輌のM365アカウント(@hanaoka-corp.co.jp)でサインインが必要です。アクセスごとにMicrosoftアカウントの選択画面が出ます。右上の「サインアウト」で抜けられます。

③ データ源と整合性

表示するすべての数字は次のSMILEデータ由来です。物理的に「FUJINだけで持っている独自データ」はありません。

  • 有効在庫一覧表.csv — 現在庫(物理・全社合算)
  • 未確定_購買手配データ.csv — 手配の中身、SMILE有効在庫(基準倉庫)
  • 確定済_工程手配一覧.csv / 確定済_購買発注一覧.csv — 確定済手配、進捗
  • 構成マスタ.csv — BOM(製番別BOM含む)
  • 品目マスタ.csv / 製番マスタ.csv — 品目情報、製番情報
  • 受注明細出力.csv / 売上明細出力.csv — 受注・売上履歴
  • 品目手順マスタ.csv — 製造工程・リードタイム

🌲 構成ツリー

🌲
構成ツリー
品目の構成・在庫・手配状態を1画面で確認するタブ

どんな時に使う

  • 出荷時に「在庫マイナス」のエラーが出た → 原因の品目を即特定したい
  • 製品の構成を見ながら、子部品の在庫がどこで足りなくなりそうか確認したい
  • BOMに「使用禁止品目」が混ざっていないか確認したい
  • マイナス在庫(物理在庫がそもそも負)になっている品目を網羅したい

使い方

1品目コードを入力して「探す」
画面上部の入力欄に品目コード(例: UDALS)を入れ、Enter または「探す」ボタン。
2構成ツリーが描画される
親方向(その品目がどの製品で使われているか)と子方向(その品目が何で構成されているか)の両方が見えます。
3右ペイン「品目詳細」で在庫・手配進捗を確認
タイルで「現在庫(物理)」「SMILE有効在庫」「発注残」「工程(未完)」「購買(未入)」が並びます。
4気になるノードをクリックすると、そのノードの詳細に切り替わる
「📍 このコードを起点にツリーを再構築」ボタンで、ツリー全体をその品目中心に再描画もできます。
5🖨 印刷用ビューボタンで「構成印刷」タブに飛ぶ
現在の品目コード・製番を引き継いで、1台あたり取数のシンプルBOM画面が開きます。

在庫タイルの読み方

品目詳細パネル内のKPIタイルは次の意味です：

✅ 現在庫(物理・全社合算)
SMILE「有効在庫一覧表」の現在庫数列。全社の物理在庫の合算値(基準倉庫だけでなく外注先支給含む全部の合計)。確認値・最優先。
SMILE有効在庫(基準倉庫)
未確定_購買手配CSVの「有効在庫数」列。引当・発注残込みのSMILE計算値、基準倉庫1か所分のみ。
発注残
確定済_購買発注一覧のうち、まだ受入されていない発注の合計。
工程(未完) / 購買(未入)
確定済手配のうち未完了/未入荷の件数。「いま動いている手配がN件ある」という意味。
FUJINで表示する現在庫はあくまで全社合算です。倉庫別・ロケーション別・仕入先支給分などの内訳が必要な場合は、SMILE「月次業務 ＞ 在庫管理帳表 ＞ 倉庫別ロケーション別品目別在庫」で確認してください。
倉庫別・ロケーション別の物理在庫は FUJIN では見られません。
FUJINで現在庫が足りなさそうに見えた場合は、SMILEの
「月次業務 ＞ 在庫管理帳表 ＞ 倉庫別ロケーション別品目別在庫」
で内訳を必ず確認してください。

状態フィルター(下部)

ツリー上のノードを状態別に絞り込めます。

  • 手配漏れ疑い: 親に需要があるのに手配がなく、現在庫も所要量未満
  • マイナス在庫: 現在庫が物理的にマイナス
  • 不足: 現在庫は0以上だが所要量に届かない
  • 依頼候補あり: 手配が必要そうな子品目を含む
  • 在庫管理対象外: SMILE品目マスタで在庫管理しない設定

⚡ 手配確定

⚡
手配確定
未確定手配にAIが判定をつけて、確定/破棄を効率化するタブ — FUJINの本丸

どんな時に使う

  • 朝のミーティング前に、今日確定すべき手配を洗い出したい
  • 過去日付の未確定手配を破棄候補かどうか見極めたい
  • 受注のある製番だけ、納期超過のものだけ、特定の判定だけを抽出したい

使い方

1上部のフィルターで見たい範囲を絞る
受注状態、納期、共通度、削除候補のみ、など多軸で絞り込みできます。
2テーブルで一覧確認
1行 = 1手配。手配予定日 / 製番 / 品目 / 数量 / 有効在庫 / 所要量 / AI判定 / 受注ラベル などが並びます。
3行をクリック → 右側に詳細パネルが開く
品目の構成ツリーが描かれ、KPIタイルで在庫・所要量を確認できます。在庫探偵タブと同じ詳細画面です。
4「📍 このコードを起点にツリーを再構築」で別の品目視点へ移動可能

受注ラベルの意味(5類型)

各手配の「受注ラベル」列には、その手配がなぜ存在するのかのSMILE的背景がついています：

ラベル意味対応のヒント
🟢 受注関連する受注が現に存在するそのまま確定すべき
🟠 期限超過納期を過ぎているが受注はまだ生きている納期調整＋優先確定
🟡 長期放置未確定のまま長期間動いていない営業に確認・破棄検討
🔵 計画放置生産計画から外れた可能性のあるもの計画見直し・破棄検討
⚫ 売上済関連受注は既に売上計上済(役目終わり)破棄候補
運用のコツ： 「⚫ 売上済」ラベルが付いた手配は、SMILE上で強制完納するか削除するのが基本。これを定期的に掃除すると未確定手配の件数が減り、本当に判断すべきものに集中できます。

🖨 構成印刷

🖨
構成印刷
BOMを「1台あたり取数」で印刷・CSV出力する辞書ツール

どんな時に使う

  • ある製品のBOMを印刷したい(現場の作業指示用)
  • 製番ごとのBOM差分を確認したい
  • 1台あたりの取数だけを純粋に知りたい(在庫情報なし)
  • CSV出力してExcelで整形したい

使い方

1品目コードと製番(任意)を入力
製番を入れると、その製番別BOMが優先表示されます(無ければ通常BOM)。
2構成ツリーが縦型で展開
レベル別に親→子と並び、各行に「1台あたりの取数」が表示されます。
3階層の畳み/展開で表示量を調整
レベルボタンで深さを揃えたり、行をクリックして個別に開閉できます。
4印刷 / CSV出力が可能
右上のボタンから。印刷時はサイドバーや余計なUIは自動で隠れます。
使用禁止品目はデフォルトで非表示になっています。SMILE品目マスタの「使用禁止日」が今日以前のものが対象。検索結果に「🚫使用禁止」バッジ付きで表示したい場合は、上部の「使用禁止品目も表示」にチェックを入れてください。
在庫情報は出ません(意図的)。 このタブはあくまで「構成の辞書」です。在庫を見たい場合は 🌲 構成ツリー タブを使ってください。構成ツリー側の 🖨 印刷用ビュー ボタンから直接ジャンプできます。

よくある質問

Q1. 現在庫の数字がSMILE画面と違うのですが？

FUJINに表示している「現在庫(物理・全社合算)」は全倉庫の合計です。SMILEで「倉庫別在庫一覧表」を見ると、第一工場・第二工場・第三工場・外注先支給などに分かれているため、見ている画面の合計が一致しないように見えることがあります。
詳細はSMILE「月次業務 ＞ 在庫管理帳表 ＞ 倉庫別ロケーション別品目別在庫」で確認してください。

Q2. データ基準日が今日でないのですが？

朝3時のRPAが何らかの理由で止まっている可能性があります。「⚠ N日前」と表示されていたらシステム担当に連絡してください。RPAが復旧すれば翌朝4時に自動で最新化されます。

Q3. AIの判定が間違っているように見える時は？

AIの判定はあくまで「判断のたたき台」です。最終判断は現場の知見が優先。気づきがあればシステム担当(雅)までフィードバックをお願いします。判定ロジックを継続改善しています。

Q4. SMILEで作業した結果はいつ反映されますか？

翌朝4時です。例えば今日18時にSMILEで手配を確定しても、FUJIN画面はまだ反映されていません(あるいは今朝3時時点のスナップショットを見続けています)。リアルタイム連携ではない点だけ留意してください。

Q5. 印刷したい / CSV出力したい

品目構成検索タブは印刷/CSV出力ボタンを備えています。他のタブは現状ブラウザの印刷機能(Cmd+P / Ctrl+P)で対応してください。要望が多ければ正式ボタンを追加します。
